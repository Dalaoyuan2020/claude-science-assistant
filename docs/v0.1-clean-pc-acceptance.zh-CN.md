# Claude Science 助手 v0.1：第二台干净电脑验收清单

本清单用于用户提供第二台 Windows 电脑后做最终验收。未完成本清单前，不把 v0.1 标记为完成。

## 1. 测试前约束

- 不在聊天、验收记录、日志、诊断包或便携包中写入、回显或提交 API Key、OAuth token、控制 token、代理凭证。
- 用户主动添加 API Key 后，当前激活 Key 会写入 WSL Bridge 运行时配置 `~/.claude-science/proxy/config.json`；必须验证该文件权限为 `0600`。
- 默认只读体检；安装 WSL、启用 Windows 功能、注册服务、启动服务或重启前必须得到用户确认。
- 不修改 Clash、v2rayN、sing-box、VPN、DNS、hosts、系统代理、根证书或 443 端口。
- 若 Windows 要求重启，记录阶段并停止；重启后重新体检再继续。

## 2. 干净机检查

1. 运行 `bootstrap-claude-science-wsl` Skill 的 Windows 只读体检。
2. 确认 Windows 10 22H2/Windows 11 x64、系统盘至少 8 GB 可用，15 GB 以上更稳。
3. 确认没有旧 Windows Bridge 进程，没有 9876/8765 端口冲突。
4. 若无 Ubuntu/WSL2，先让 Skill 输出安装计划。

## 3. 安装/修复流程

1. 解压便携 ZIP 后，在解压目录根部执行以下命令；不要只复制单个 exe，因为启动器和 Skill 需要同目录下的 `proxy.py`、`requirements.txt`、`scripts/`、`static/`、`tests/`、`docs/` 和 `skills/`。

2. 推荐先双击包根目录的 `1-run-acceptance-preview.bat`。它会检查包结构、跑 Windows 只读体检、repair PlanOnly 和 rollback PlanOnly；默认不安装、不启动、不删除。

   如果需要命令行方式，也可以运行保留的 PowerShell 原版：

   ```powershell
   powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts\acceptance-v0.1.ps1
   ```

3. 推荐在预览后双击包根目录的 `2-collect-acceptance-evidence.bat` 生成一份脱敏证据包，方便回传问题而不暴露 API Key。该入口仍然只做只读/预览检查，不执行安装或删除；生成的 zip 位于 `acceptance-evidence\`。

   命令行备用方式：

   ```powershell
   powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts\collect-acceptance-evidence.ps1
   ```

   如果只想收集已存在的体检状态、不重复运行完整验收预览，可以加 `-SkipAcceptancePreview`。

4. 如果需要单独运行 Skill，可以运行：

   ```powershell
   powershell.exe -NoProfile -ExecutionPolicy Bypass -File skills\bootstrap-claude-science-wsl\scripts\repair-approved.ps1 -ProjectRoot . -PlanOnly
   ```

5. 用户确认后，再通过验收助手执行安装、启动和自测：

   普通用户推荐双击包根目录的 `4-install-runtime-after-preview.bat`，它会再次询问确认，然后安装/修复 WSL runtime、安装包内置的锁定版 Claude Science Linux 二进制、启动服务并运行自测。

   命令行备用方式：

   ```powershell
   powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts\acceptance-v0.1.ps1 -ApproveInstall -StartServices -RunSelfTest
   ```

   也可以直接运行底层 Skill：

   ```powershell
   powershell.exe -NoProfile -ExecutionPolicy Bypass -File skills\bootstrap-claude-science-wsl\scripts\repair-approved.ps1 -ProjectRoot . -ApproveInstall -StartServices -RunSelfTest
   ```

6. 只有确实需要安装 Ubuntu 时，才额外加入 `-InstallWslIfMissing`，并使用管理员 PowerShell。

7. 环境预览和安装完成后，可双击包根目录的 `3-open-claude-science-assistant.bat` 打开启动器；也可以直接双击 `claude-science-assistant.exe`。

## 4. 启动器验收

- 启动器能打开，标题为“Claude Science 助手”。
- 首页状态能显示 WSL2、Bridge、Claude Science、当前 API Key。
- 启动、停止、重启、打开 Claude Science 不产生重复 Bridge。
- “配置面板”按钮能在 Bridge 运行时打开本地 dashboard；若启用 required path secret，不在启动器界面明文显示 token。
- 首页不平铺所有 Provider；API Key 按添加顺序排列，一次只突出一条当前 Key，并提供“添加 API Key”入口。
- 点击“添加 API Key”后，服务商模板顺序为：
  1. GLM-5.2、LongCat、DeepSeek、Claude、OpenAI/GPT
  2. OpenCode Go、OpenRouter
  3. 内置中转、自定义中转
- 内置中转显示 `https://10521052.xyz/v1`，但不标记为官方或可信。
- 非 Claude 官方登录项添加时必须输入 Key；保存后列表显示“已加密保存”，但不回显 Key 或密文。
- 当前激活 Key 应只出现在 WSL Bridge 运行时配置中，配置文件权限为 `0600`；前端、日志和验收输出不得出现明文 Key。
- 至少添加两条测试 Key 后，点击“使用”可切换当前 Key；当前 Key 的“删除”按钮禁用，切换后方可删除。
- 自定义中转默认留空，第三方中转未确认时不应用到 Bridge。

## 5. 通过标准

- Windows 体检：`duplicate_bridge_warning=false`，`secrets.values_included=false`。
- WSL 体检：`bridge_healthy=true`，`port_9876=true`，`port_8765=true`。
- 项目自测：`self-test passed`。
- 便携包内置 Claude Science Linux 二进制：`vendor\claude-science\linux-x64\claude-science` 存在，manifest 中的版本和 SHA256 可核对。
- 便携包：`scripts/package-launcher-portable.ps1` 生成 zip 和 `.sha256`。
- BAT 双击入口：包根目录的 `1-run-acceptance-preview.bat`、`2-collect-acceptance-evidence.bat`、`3-open-claude-science-assistant.bat`、`4-install-runtime-after-preview.bat` 可用；`.ps1` 原版仍保留。
- 证据包：`2-collect-acceptance-evidence.bat` 或 `scripts/collect-acceptance-evidence.ps1` 能生成脱敏 zip；分享前仍需人工确认其中没有 Key 或 token。

## 6. 失败处理

- 端口冲突：只停止确认属于本项目的旧进程。
- WSL 缺包：按脚本提示在 Ubuntu 中安装 `python3-venv`、`python3-pip`、`curl` 后重跑。
- Tauri MSI/NSIS 打包失败：记录 WiX/NSIS 下载或安装器工具链错误；v0.1 可先用便携包测试，正式安装器作为发布工程修正项。

## 7. 卸载/回滚验证

默认先预览，不删除任何东西：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File skills\bootstrap-claude-science-wsl\scripts\rollback-approved.ps1 -ProjectRoot . -PlanOnly
```

用户明确确认后再执行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File skills\bootstrap-claude-science-wsl\scripts\rollback-approved.ps1 -ProjectRoot . -ApproveUninstall
```

通过标准：WSL user service 被禁用并移除；产品管理的 patched copy 和 venv 被移除；Provider config、API Key、OAuth token、原始 `~/.local/bin/claude-science` 保留。`-DeleteProductData` 必须单独确认。
