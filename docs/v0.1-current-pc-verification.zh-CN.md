## 4. Provider 实测

测试只输出状态和能力布尔值，不记录 Key 或回答正文。`32768` 是低成本参数接受测试，不是官方精确最大输出。

| Provider | 模型列表 | 短文本 | 接受 32768 | 函数调用 | Reasoning |
| --- | --- | --- | --- | --- | --- |
| OpenCode Go / `glm-5.2` | 200，20 个模型 | 通过 | 是 | 通过 | `thinking:{type:"enabled"}` 通过 |
| 已保存自定义中转 / `claude-opus-4-8` | 200，4 个模型 | 通过 | 是 | 通过 | 未在未知 relay 协议上强制注入 |
| 内置中转 / `gpt-5.5` | 502 | 502 | 无法判断 | 无法判断 | 无法判断 |
| DeepSeek 官方 / `deepseek-v4-pro` | `/models` 未返回可测试模型 | 流式与非流式均 200 | 未探测 | 未探测 | `auto` 转 `adaptive` 后通过 |
| MiniMax 中国区官方 | 能力探测 | 基础对话可用 | 未探测 | 未探测 | 未完整探测 |

内置中转的 502 表示上游在验证时不可用，不能据此判断模型输出或工具上限。两个可用 OpenAI-compatible `/models` 响应都没有给出 `max_tokens`、`max_output_tokens` 或 `context_window`，因此 Bridge 不伪造精确上限。

### 2026-07-12 当前运行实例复验

- 状态探测：`overall=ready`、`bridge_healthy=true`、`bridge_source_matches=true`、`unit_matches_project=true`，无 warnings。
- 接管前实例 PID：Bridge `150785`，Claude Science `150869`；候选包接管后为 Bridge `152400`、Claude Science `152482`。PID 只作为本次现场证据，不作为跨机器固定值。
- 接管后候选包和最终包的脚本状态探测均返回 `source_matches=true`、`unit_matches_project=true`、无 warnings。
- 发布后从最终目录启动 GUI 时，观察到 GUI 提示“another or older package”，但同目录 `status-probe.ps1` 报告来源匹配。r2 诊断确认 expected/actual 只是 Windows DrvFs 路径大小写不同；改为仅对 `/mnt/<盘符>/` 大小写不敏感比较后，真实 r2 EXE 六状态全部恢复健康。
- 非流式脱敏对话：HTTP 200，返回 Anthropic `message` 结构，包含 1 个内容块。
- 流式脱敏对话：HTTP 200，`Content-Type=text/event-stream`，事件链包含 `message_start`、内容块事件、`message_delta` 和 `message_stop`。
- 两次请求均携带 `thinking.type=auto`，证明 Bridge 的 DeepSeek `auto -> adaptive` 兼容转换在真实运行实例中生效。
- 验收只记录状态码、事件类型和结构布尔值，没有记录 API Key 或模型回答正文。
