# CSA v0.1.1 当前电脑验证记录

验证范围：当前开发电脑上的源码、启动器、WSL 运行时、便携包预览流程和文档清单。  
注意：本记录不能替代第二台干净 Windows 电脑验收；干净机验收仍见 [v0.1-clean-pc-acceptance.zh-CN.md](v0.1-clean-pc-acceptance.zh-CN.md)。

## 1. 产品逻辑核对

| 要求 | 当前证据 | 结论 |
| --- | --- | --- |
| 启动器不是 Provider 平铺面板 | 首页只展示当前 Key、状态和启动动作；Provider 模板只在“添加 API Key”流程出现 | 通过 |
| API Key 按添加顺序排列 | `launcher/src/App.tsx` 渲染保存列表；后端测试覆盖顺序保持 | 通过 |
| 一次只激活一条 Key | 当前 Key 高亮；切换通过 `activate_api_key` 完成；当前 Key 删除受限 | 通过 |
| 第三方中转不默认信任 | 内置中转和自定义中转均归入第三方；未确认域名前不应用 Bridge 配置 | 通过 |
| API Key 不回显 | 前端只显示是否已加密保存；后端状态不返回明文或密文 | 通过 |
| 当前激活 Key 可写入 WSL Bridge | 后端会生成 WSL 运行时配置并设置 `0600` 权限 | 当前电脑通过代码与本机验证；仍需干净机实测 |

## 5. 便携包检查

推荐发布资产：

```text
dist\claude-science-assistant-v0.1.1-release-portable.zip
dist\claude-science-assistant-v0.1.1-release-portable.zip.sha256
```

包内必须包含：

- `claude-science-assistant.exe`
- `manifest.json`
- `proxy.py`
- `requirements.txt`
- `scripts/`
- `static/`
- `tests/`
- `docs/`
- `skills/`
- `vendor/claude-science/linux-x64/claude-science`

不要只分发 exe；启动器需要同目录下的脚本、文档、Skill、Bridge 和内置运行时。

## 6. 未完成项

v0.1.1 可以作为首个 GitHub 公开版本发布，但最终“干净机完成”仍需第二台 Windows 电脑验证：

1. 解压 release 便携包。
2. 双击 `1-run-acceptance-preview.bat`。
3. 生成脱敏证据包并人工确认没有 Key/token。
4. 用户确认后运行安装/启动/自测流程。
5. 添加至少两条非 Claude 测试 API Key，验证顺序、切换、删除限制、第三方确认和不回显。
6. 验证 WSL Bridge 运行时配置权限为 `0600`。
7. 验证没有重复 Bridge。
8. 运行 rollback PlanOnly，必要时验证卸载只删除产品管理内容。
