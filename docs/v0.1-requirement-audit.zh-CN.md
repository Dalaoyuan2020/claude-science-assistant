日期：2026-07-06

本文把当前用户目标拆成可核验条目，避免范围漂移。状态含义：

| 状态 | 含义 |
| --- | --- |
| 已验证 | 当前代码、文档、测试或运行证据足以支撑 |
| 待干净机验证 | 当前电脑已有实现和本机证据，但仍需第二台干净 Windows 电脑复验 |

## 7. 首发仍需注意

- v0.1.1 可以作为第一版推上 GitHub，但文案要明确“首个公开版本”，不要承诺任何电脑一键成功。
- 第二台干净 Windows 电脑验收仍是最终产品完成条件。
- 发布 Release 时，用最新 `.sha256` 文件填写 Release 页面，不要把旧 SHA 写死进会被打包的文档。
- 任何 issue、截图、日志、证据包都不能包含真实 API Key。
