## 1. 产品与界面

- 启动器版本统一更新为 v0.1.3。
- WSL、运行时、Bridge、Claude Science、WSL 存储和当前 API Key 组成 2 行 × 3 列六项状态栏。
- 状态栏支持折叠，折叠状态不影响后台刷新和诊断。
- WSL 存储状态显示 VHDX 的 Windows 位置、宿主盘余量、Linux 根分区余量和 VHDX 大小。
- 自定义中转支持用户命名；留空时生成“自定义中转 YYYY-MM-DD #NN”。
- Provider 初始模型保持为空，不再内置钉死 GLM 或其他默认模型。
- API Key 列表按添加顺序展示，同时只激活一条配置。

## 2. 启动器稳定性

- WSL 和 PowerShell 工作移出 UI 主线程并设置超时，避免按钮操作导致窗口长期冻结。
- 状态探测合并为一次只读检查，减少启动等待和重复拉起开销。
- 修复启动脚本把失败误报为成功的问题。
- Bridge 健康判断不再依赖必须命中 PID 正则；健康响应可作为独立证据。
- Bridge 健康接口增加 `source_path` 和 `config_revision`，防止旧目录实例冒充当前版本。
- 新包可识别旧目录 Bridge，并按候选包接管流程停止旧实例、启动当前实例和验证来源。
- 非 systemd fallback Bridge 可脱离启动 shell 稳定运行，并要求连续健康检查通过。

## 3. Provider、模型和协议

- “添加 API Key”保持为 Provider 模板的统一入口，不把所有 Provider 平铺到首页。
- 自动映射支持单模型全部角色映射，以及多模型主力/快速评分。
- OpenCode Go 使用实时模型列表评分，不向账户写入没有返回的模型。
- MiniMax 使用中国区 Anthropic 兼容地址 `https://api.minimaxi.com/anthropic`。
- DeepSeek 使用原生 Anthropic 兼容地址 `https://api.deepseek.com/anthropic`。
- DeepSeek `/models` 不返回可测试列表时，可尝试经过资料核验的官方候选；候选只有真实短对话成功后才保存。
- 修复 DeepSeek/MiniMax 原生接口不接受 `thinking.type=auto` 的问题：出站请求转换为 `adaptive` 并移除固定 `budget_tokens`，显式 `enabled` / `disabled` 保持不变。
- OpenAI o-series 使用 `max_completion_tokens`，并保留并行工具调用偏好。
- 正式对话不注入统一输出长度；调用方未提供预算时，由上游模型使用自身默认值。
- 只有明确 HTTP 400/422 参数错误才允许一次兼容重试；认证、额度、模型、网络和 5xx 不伪装成参数问题。

## 4. Key 切换与安全

- API Key 使用 Windows 当前用户 DPAPI 加密保存。
- Key 切换采用“预写 Windows 设置 → 更新 WSL 配置 → 重启 Bridge → 验证配置版本 → 原子提交”的事务流程。
- 切换失败时恢复 Windows 设置和 Bridge 配置。
- 诊断、日志、文档和发布包不回显明文 Key。
- 项目方自建中转与自定义中转都必须由用户确认目标域名；不会使用官方 Provider 的模型候选回退。

## 5. WSL 与存储防护

- 推荐 Ubuntu-24.04，同时兼容已有可用 Ubuntu/WSL2 环境。
- 检测 `/tmp`、用户目录、根文件系统只读状态、磁盘余量和 inode。
- 文件系统异常时停止自动重启，避免扩大损坏。
- Bridge 日志超过 50 MB 时轮转自身日志，并只保留一份备份。
- 当前版本只提供迁移诊断和计划，不自动注销、迁移、压缩或重建 WSL 发行版。

## 6. 安装与升级

- 公开包同时保留 BAT 和 PowerShell 入口。
- 新增首次安装 / 同用户升级通用 AI Prompt，由 AI 先只读判断场景，再等待用户批准。
- 首次安装没有 WSL 时，必须单独说明管理员权限、Windows 功能和重启要求。
- 同一电脑、同一 Windows 用户升级时，不卸载 WSL/Ubuntu，不清空 APPDATA，不要求重新添加已保存 Key。
- 升级必须下载完整 ZIP 并解压到新目录；不能只替换 EXE。
- 新版接管成功并完成真实对话前，保留旧目录作为回退。

## 9. 发布内容与排除内容

源码仓库应包含：公开源码、测试、脚本、Skill、README、Release 文案、Provider 矩阵、更新记录和公开技术计划。

GitHub Release 应包含：

- `claude-science-assistant-v0.1.3-release-portable.zip`
- `claude-science-assistant-v0.1.3-release-portable.zip.sha256`

不得提交：

- `config.json`、`.env`、用户设置、日志和诊断包；
- `private/` 与付费用户 HTML；
- `dist/`、Rust `target/`、`target-validation/`、Python venv 和缓存；
- 明文 API Key、OAuth token、Cookie、控制 token；
- Claude Science Linux 二进制到源码 Git 历史。该二进制只进入 Release ZIP。
