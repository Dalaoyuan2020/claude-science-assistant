## 3. 功能实例

### 3.1 Connect 文本闭环

```text
Telegram 私聊
-> Gateway 授权与去重
-> 持久队列
-> 浏览器插件投递当前 Claude Science 页面
-> Claude Science 回复
-> Gateway 回到原 Telegram chat/thread
```

预期：同一 `platformEventId` 重放不会生成第二个待处理消息；`delivery_unknown` 不自动重试，避免重复发送。

### 3.2 Connect 图片闭环

```text
Claude Science 生成 artifact
-> 只读解析受管目录
-> MIME/魔数/大小/SHA-256 校验
-> Telegram sendPhoto/sendDocument
-> 写入平台消息 ID
```

自动媒体测试已通过。历史真人测试曾返回 Telegram 图片消息 ID；RC 发布前仍需按第 5 节复验一次。

### 3.3 Subagent 沙盒外调用

```text
Claude Science Skill
-> reports/csa-agent-inbox/<requestId>.json
-> CSA 面板显示并等待人工批准
-> 本机 Claude Code headed plan 会话
-> reports/csa-agent-runs/<runId>/
-> reports/csa-agent-outbox/<requestId>.json
-> Claude Science 读取摘要继续工作
```

外部 Agent 默认只做诊断、计划或审查。安装、删除、迁移、网络配置等动作不会从沙盒请求中自动执行。

## 4. 本轮修复的发布阻断项

1. 增加稳定 outbox，Claude Science 不再猜测哪个 run 是最新结果。
2. outbox 只返回项目内相对路径，并对摘要做凭据样式脱敏。
3. Subagent 请求脚本拒绝明显 API Key、Token、密码和私钥内容。
4. 增加统一新功能验证器和 JSON 证据。
5. 新增 CSA External Agent Skill，新装和修复流程可自动部署。
6. 便携包纳入 Gateway、两个 Skill、浏览器插件、请求脚本、验证脚本和新版文档。

## 5. 待人工复验清单

按顺序执行，预计 10 分钟：

1. 打开 CSA 和 Claude Science 页面，Connect 显示 Gateway、页面、回复连接均就绪。
2. 从本人 Telegram 私聊发送唯一文本，例如 `RC1-TEXT-<当前分钟>`。
3. 确认消息只进入当前项目一次，并且 Telegram 只收到一份最终回复。
4. 让 Claude Science 生成一张小型 PNG，并明确要求通过当前 Connect 请求回复。
5. 确认 Telegram 收到可打开的图片，文字说明不重复，面板历史只有一条完成记录。
6. 在 Subagent 写入 Demo，批准运行，终端返回后确认 outbox 出现；恢复同一 session 追问一次，确认有上下文。

任一消息进入 `delivery_unknown` 时不要手动重复点击发送。先在只读历史中确认 Claude Science 是否已经收到，再决定人工重发。

## 6. 已知边界

- 第一版只承诺本人私聊和纯文本主链路；附件入站、群聊与团队角色后续扩展。
- 飞书企业自建应用仍需真实租户权限与人工验收。
- 浏览器插件是首选投递层，UIAutomation 与队列是回退层。
- Claude Science 空闲时不会被 Skill 主动唤醒；浏览器页面就绪时可由插件触发。
- 当前证据来自本机、当前 Windows 用户和 `Ubuntu-24.04`；新电脑仍需独立安装验收。
