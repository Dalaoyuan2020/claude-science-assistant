# CSA UI 与 Subagent 联合验收案例

## 1. 本轮范围

- 四模块统一视觉基座与窗口比例。
- Subagent 任务切换、Claude Code CLI 恢复和聊天记录同步。
- Connect 仅验证配置界面与安全边界，不发送真实飞书或 Telegram 消息。
- Research OS 的业务验收推迟到知识飞轮方案讨论完成后。

## 2. 测试前置条件

1. 使用默认窗口 `1040 x 820` 启动 CSA。
2. 本机已安装 Claude Code CLI，并至少存在两个 CSA Subagent 任务。
3. 其中一个任务已经生成 Claude Code `session_id`。
4. 测试过程中不填写真实 API Key、Webhook 或 Bot Token 到截图和测试记录。

## 3. UI 验收

| ID | 操作 | 预期结果 |
|---|---|---|
| UI-01 | 依次切换 API Key、Subagent、Connect、Research OS | 四个页面使用一致的白色工作区底框，页面宽度不跳动 |
| UI-02 | 在默认窗口查看四个页面 | 页面无横向滚动条，按钮和文字无重叠 |
| UI-03 | 在 `1040 x 820` 查看 Subagent | 整页高度不超过视口，聊天记录在内部滚动 |
| UI-04 | 将窗口缩到最小尺寸 `760 x 640` | 无横向溢出；允许出现整页纵向滚动 |
| UI-05 | 展开和收起环境状态 | 模块导航与工作区不发生横向位移 |
| UI-06 | 展开诊断信息 | 诊断内容在底框上方展开，不遮挡导航或工作区 |
| UI-07 | 切换模块后重启 CSA | 恢复上一次选中的模块 |
| UI-08 | 查看 Connect | 飞书和 Telegram 显示为两张等高连接卡，状态和操作层级清楚 |
| UI-09 | 查看 Research OS 空状态 | 仓库输入区与空状态边界明确，不出现大片无结构空白 |
| UI-10 | 检查键盘焦点 | Tab 键可以依次访问模块、主要按钮和表单输入，焦点清晰可见 |

## 4. Subagent 核心验收

| ID | 操作 | 预期结果 |
|---|---|---|
| SA-01 | 选择任务 A，再选择任务 B，再切回任务 A | 任务 A 的运行记录和聊天记录仍然存在 |
| SA-02 | 点击“在 Claude Code 中继续” | 打开外部终端并执行当前 `session_id` 对应的 `claude --resume` |
| SA-03 | 在外部 CLI 回复一句话 | CSA 在 2 至 5 秒内显示新消息，无需重新选择任务 |
| SA-04 | 连续在 CLI 对话五轮 | CSA 始终显示同一会话上下文，消息顺序为最新在前 |
| SA-05 | 聊天记录超过 220px | 只有聊天记录区出现滚动，任务详情和整个页面高度不继续增长 |
| SA-06 | 点击“加载更早记录” | 每次增加读取数量，最多读取 200 条；现有记录不丢失 |
| SA-07 | 切换到另一个任务再切回 | 内部滚动区重新显示该任务的真实 session，而不是上一个任务的记录 |
| SA-08 | 选择没有 session 的任务 | 显示明确空状态，不显示错误任务的历史记录 |
| SA-09 | session 文件包含工具调用或附件事件 | 只展示用户和助手文本，不展示工具事件原始 JSON |
| SA-10 | session 文本包含常见密钥字段 | 敏感行被脱敏，不在 CSA 界面显示完整值 |
| SA-11 | Claude CLI 不存在或启动失败 | 显示可理解的失败原因，任务和历史记录不被清空 |
| SA-12 | 快速连续切换任务 | 不发生记录串线、重复请求导致的界面闪烁或状态覆盖 |

## 5. Connect 界面与边界验收

| ID | 操作 | 预期结果 |
|---|---|---|
| CN-01 | 打开飞书配置 | 只显示官方 Webhook 输入，不显示远程命令或聊天入口 |
| CN-02 | 打开 Telegram 配置 | 显示 Bot Token 和 Chat ID；Token 使用密码输入框 |
| CN-03 | 未填写必要字段 | “加密保存”保持禁用 |
| CN-04 | 保存后重新打开配置 | 密钥不回显，只显示“已加密配置”的摘要 |
| CN-05 | 查看页面说明 | 明确写明当前只出站通知、不监听消息、不接受远程命令 |

## 6. 通过标准

- `UI-01` 至 `UI-10` 全部通过。
- `SA-01`、`SA-02`、`SA-03`、`SA-05`、`SA-07` 为阻断项，任一失败都不能判定 Subagent 可用。
- Connect 真实发送测试暂不作为本轮通过条件。
- Research OS 测试案例必须等待知识飞轮的数据模型、摩擦分类和 Skill 生命周期确定后再编写。
